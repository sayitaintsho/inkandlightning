# Ink & Lightning - Agent Guide

## Project
- **What:** Static website for Ink & Lightning (founder-led marketing)
- **Hosting:** GitHub Pages from `main` branch
- **Repo:** git@github.com:sayitaintsho/inkandlightning.git
- **Stack:** Pure HTML/CSS (no build step)

## Files
- `index.html` — Main landing page
- `about.html` — About page
- `README.md` — Repo readme

## Workflow
1. Make changes to files in `/Users/dan/code/inkandlightning`
2. `git add . && git commit -m "description" && git push origin main`
3. GitHub Pages auto-deploys from main

## Guidelines
- Keep it simple — static HTML/CSS, no frameworks
- Preserve the existing design aesthetic (serif fonts, cream/ink palette)
- Test changes make sense before committing
- Commit with descriptive messages
- Always push after committing so GitHub Pages updates
